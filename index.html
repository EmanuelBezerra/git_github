Anotações do Curso:

  • Para iniciar o git, precisa clicar com o botão direito na pasta desejada, clicar em 'git bash here', e digitar 'git init'.
  • Para aparecer a pasta '.git', precisa ir na opição 'exibir no canto superior esquerdo, e clicar na opção 'itens ocultos' no canto superior direito.
  • Para saber a situação atual do git, digite 'git status', e vai aparecer as informações, como por exemplo, arquivos que não foram adicionados.
  • Para adicionar arquivos, digite 'git add "nome do arquivo"'., caso queria add tudo de uma vez, digite 'git add .'.
  • Para validar os arquivos adicionados, precisa digitar 'git commit -m "explique brevemente aqui as alterações"'.
  • O '-m' significa mensagem.
  • Caso peça seu nome e e-mail, é só adicionar usando o texto que eles dão.
  • Para adicionar o local para onde vai os arquivos, precisa digitar 'git remote add origin (coloca o https ou ssh que encontra no github aqui, sem os Parênteses)'.
  • Para ver para onde os arquivos estão indo, digite 'git remote -v'.
  • para remover o local para onde o arquivo vai ser enviado, digire 'git remote remove origin'.
  • Para enviar para o github, precisa digitar 'git push'.
  • Outra forma para conseguir criar um git, é copiando o https ou ssh no github, ir na pasta, abrir o git bash here, e digitar 'git clone (coloca o https ou ssh)'.
  • Digitando 'git reflog', você consegue ver o historico de todas as versões do seu codigo, mostrando o nome dado, e o id da versão.
  • Para migrar entre as versões do seu codigo, precisa digitar 'git reset --hard (coloque o id da versão do codigo aqui)'.
  • Uma Branch é basicamente um dos lugares onde vai salvar, podendo ter bem mais de um save, e você pode migrar entre eles, e juntar eles também.
  • Digitando 'git branch' vai mostrar quantas branchs tem nesse git.
  • Para criar uma nova branch, digite 'git branch staging', e a palavra stanging vai ser o nome da branch, pode ser outro nome se quiser.
  • Para migrar entre branchs, digite 'git checkout staging', sempre colocando o nome da branch no final para ir para ela.
  • Para trazer a atualização de outra branch, para a atual, digite 'git merge staging', usando o nome da branch que vai ser mesclada.
  • É aconselhavel usar o comando 'git pull' para deixar a branch na atualização mais atual, para evitar problemas quando estiver trabalhando em equipe.
